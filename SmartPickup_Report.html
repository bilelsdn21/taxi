Fonctionnalités Implémentées
✅ Auth complète

Login JWT avec sauvegarde token (FlutterSecureStorage), navigation par rôle (driver → /driver, user → /userlayout), Signup, bypass dev pour tests sans token.

✅ Dashboard Chauffeur

Affichage courses disponibles, course active en cours, profil chauffeur, toggle Online/Offline, Accept/Reject course avec navigation automatique.

✅ Ride Requests

Liste des courses pending, accordéon détails, boutons Accept/Reject, navigation vers Active Ride après acceptation.

✅ Active Ride + Carte

FlutterMap + tuiles Mapbox Navigation Night, marqueurs pickup/destination/taxi animé, polyline itinéraire, rotation taxi selon heading GPS.

✅ WebSocket GPS

Connexion WS persistante au backend, publication position toutes les 2s, stockage en BDD (Taxi.latitude/longitude/last_updated), reconnexion auto.

✅ Mapbox Directions

Itinéraire réel recalculé toutes les 6s, affichage distance + durée estimée, deux phases : vers pickup puis vers destination finale.

✅ Mode Simulation

Mouvement simulé vers la destination (test sans GPS physique), toggle via bouton robot sur la carte.

✅ Historique Courses

Liste triée par date avec statut (completed / cancelled), durée, pickup/dropoff, rating par défaut.

⚠️ Profil Chauffeur

Page UI existante, endpoint GET/PUT profile fonctionnel, mais upload photo non encore branché côté frontend.

🔵 Side Passager

Écrans BookRide, TrackRide, Schedule, Incident, RideHistory présents. Connectivité backend partielle.

Bugs Corrigés Aujourd'hui
FIX 1 Dashboard — Nom chauffeur ne s'affichait pas
📄 driver_dashboard.dart · ligne 166

L'accès au profil utilisait _driverProfile!['user']['full_name'] mais le backend retourne full_name à la racine, pas imbriqué dans une clé user. Résultat : le nom s'affichait toujours "Driver". 

- _driverProfile!['user']['full_name']
+ _driverProfile!['full_name']
FIX 2 Login — URL hardcodée, token non sauvegardé, mauvaise navigation
📄 login.dart

Trois problèmes en un : (1) URL pointait vers une IP privée fixe (10.122.164.121) au lieu de localhost dynamique, (2) le JWT access_token n'était jamais sauvegardé en secure storage, (3) tous les utilisateurs étaient redirigés vers /userlayout même les chauffeurs. 

- Uri.parse('http://10.122.164.121:8000/auth/login')
+ Uri.parse('$_baseUrl/auth/login')  // localhost adaptatif

- // token non sauvegardé
+ await _storage.write(key: 'token', value: data['access_token']);
+ await _storage.write(key: 'role', value: data['role']);

- Navigator.pushReplacementNamed(context, '/userlayout');
+ final route = data['role'] == 'driver' ? '/driver' : '/userlayout';
+ Navigator.pushReplacementNamed(context, route);
FIX 3 DriverRequests — Navigation manquante après acceptation
📄 driver_requests.dart

Après avoir accepté une course depuis l'onglet Requests, le chauffeur restait sur la même page. Le Dashboard naviguait correctement vers /driver/active, mais Requests ne le faisait pas. 

+ Navigator.pushReplacementNamed(context, '/driver/active');
FIX 4 Active Ride — Overlay coordonnées GPS défini mais jamais affiché
📄 driver_active_ride.dart · méthode _buildMap()

La méthode _buildCoordinatesOverlay() était entièrement implementée (widget avec lat/lng en temps réel, bordure jaune) mais n'était jamais ajoutée au Stack de la carte. 

+ _buildCoordinatesOverlay(),  // ajouté dans le Stack
AMÉLIORATION DriverApiService — Token maintenant envoyé dans les requêtes
📄 driver_api_service.dart

Les headers d'authentification étaient commentés. Maintenant le token JWT est lu depuis le secure storage et envoyé automatiquement si présent. Le bypass backend (auto-login du premier driver) reste actif pour les tests sans token. 

- // String? token = await _storage.read(key: 'token');
- // 'Authorization': 'Bearer $token',
+ final String? token = await _storage.read(key: 'token');
+ if (token != null) 'Authorization': 'Bearer $token',
Flow Complet du Chauffeur
1. Login → /driver
JWT sauvegardé, role détecté, redirection vers Dashboard chauffeur.
2. Dashboard — Voir les courses disponibles
Polling de GET /api/driver/rides/available. Requests affichées avec accordéon détails.
3. Accepter une course
POST /api/driver/rides/{id}/accept → RideAssignment créé en BDD → Navigation automatique vers Active Ride.
4. Active Ride — Localisation GPS
Démarrage du tracking GPS (Geolocator), WebSocket /ws/location/{id} ouvert, position envoyée toutes les 2s, stockée en BDD (Taxi table).
5. Carte Mapbox — Itinéraire dynamique
Taxi animé (rotation selon heading), polyline jaune vers pickup, recalcul toutes les 6s via Mapbox Directions API, overlay coordonnées en temps réel.
6. START RIDE
PUT /api/driver/rides/{id}/status avec action=start → RideLog créé, RideEvent "started" enregistré → itinéraire bascule vers destination finale.
7. COMPLETE RIDE
action=complete → RideLog.status = completed, RideRequest.status = completed, RideEvent "completed" → retour en mode roaming.
8. Historique
GET /api/driver/rides/history → liste triée par date avec statut, passager, durée.

Ce Qui Reste à Faire
TODO Mapbox Token

Le token Mapbox est passé via --dart-define=MAPBOX_TOKEN=pk.xxx ou la valeur par défaut YOUR_MAPBOX_TOKEN_HERE est utilisée. Sans token valide, la carte et les itinéraires ne fonctionnent pas. Créer un compte Mapbox et remplacer la valeur.

TODO Passager — pickup_lat/lng

Les coordonnées GPS de pickup et dropoff doivent être capturées lors de la création d'une ride request (BookRide). Si elles sont nulles, la carte n'affiche pas les marqueurs et l'itinéraire ne se calcule pas.

TODO Photo de profil

L'upload photo est présent côté backend (dossier uploads/), mais la page profil chauffeur côté Flutter n'envoie pas encore la photo via multipart/form-data.

TODO Notifications temps réel

Le modèle Notification existe en BDD et le badge rouge "3" est hardcodé dans DriverLayout. Il faut brancher un endpoint GET /api/notifications et afficher le vrai compte.

TODO Logout

Le bouton logout dans DriverLayout et UserLayout ne fait rien (onPressed: () {}). Il faut supprimer le token du secure storage et naviguer vers /.

TODO Paiement

Le modèle Payment existe mais aucun endpoint de paiement n'est exposé. Le montant de course n'est pas calculé ni affiché.

Comment Lancer le Projet
🖥️ Backend
cd ssss/backend

# Install deps
pip install -r requirement.txt

# Create DB tables + seed data
python create_tables.py
python seed_db.py

# Start server
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# Docs disponibles sur :
# http://localhost:8000/docs
📱 Frontend Flutter
cd ssss/frontend

# Install packages
flutter pub get

# Run with Mapbox token
flutter run \
  --dart-define=MAPBOX_TOKEN=pk.YOUR_TOKEN_HERE

# Sans token (carte non fonctionnelle mais UI OK)
flutter run

# Pour Windows desktop
flutter run -d windows
